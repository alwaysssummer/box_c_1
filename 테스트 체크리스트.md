# 📋 테스트 체크리스트

## 🔗 테스트 접속 URL
```
http://localhost:3000
```

## 🏠 1. 교재관리 탭

### 1.1 그룹 관리
| 상태 | 테스트 항목 | 경로 | 설명 |
|:----:|------------|------|------|
| ☐ | 그룹 추가 | 좌측 사이드바 | `+ 그룹 추가` 버튼 → 이름 입력 → 등록 |
| ☐ | 그룹 선택 | 좌측 사이드바 | 그룹 클릭 시 선택 표시 및 중앙 패널 업데이트 |
| ☐ | 그룹 순서 변경 | 좌측 사이드바 | 드래그 핸들(⋮⋮)로 순서 변경 |
| ☐ | 그룹 삭제 | 좌측 사이드바 | 그룹 선택 → 삭제 버튼 |

### 1.2 교재 트리 (TextbookTree)
| 상태 | 테스트 항목 | 경로 | 설명 |
|:----:|------------|------|------|
| ☐ | 그룹 토글 (확장/축소) | 중앙 패널 | 그룹 노드의 ▶/▼ 클릭 |
| ☐ | 교재 토글 (확장/축소) | 중앙 패널 | 교재 노드의 ▶/▼ 클릭 |
| ☐ | 단원 토글 (확장/축소) | 중앙 패널 | 단원 노드의 ▶/▼ 클릭 |
| ☐ | 교재/단원 순서 변경 | 중앙 패널 | 드래그 핸들(⋮⋮)로 순서 변경 |
| ☐ | 지문 선택 | 중앙 패널 | 지문 클릭 시 우측 패널에 상세 표시 |

### 1.3 문장분리 (구글시트 연동)
| 상태 | 테스트 항목 | 경로 | 설명 |
|:----:|------------|------|------|
| ☐ | 시트 URL 입력 | 중앙 패널 `문장분리` 탭 | 구글시트 URL 입력 |
| ☐ | 시트 데이터 조회 | 중앙 패널 | `불러오기` 버튼 클릭 |
| ☐ | 체크박스 선택 | 중앙 패널 | 전체선택/개별선택 |
| ☐ | 문장분리 실행 | 우측 패널 `실행` 버튼 | 선택된 지문의 문장 분리 |
| ☐ | 분리 결과 확인 | 우측 패널 | 분리된 문장 목록 표시 |

### 1.4 원큐 문제 생성 (문제출제 탭)
| 상태 | 테스트 항목 | 경로 | 설명 |
|:----:|------------|------|------|
| ☐ | 탭 전환 | 중앙 패널 상단 | `문제출제` 탭 클릭 |
| ☐ | 지문 계층 선택 | 좌측 트리 | 교재/단원/지문 체크박스 선택 |
| ☐ | 상위 선택 시 하위 자동 선택 | 좌측 트리 | 교재 체크 → 하위 모든 지문 선택 |
| ☐ | 문제 유형 선택 | 우측 패널 | 드롭다운에서 문제 유형 선택 |
| ☐ | 자동 사전 검증 | 우측 패널 | 유형 선택 시 자동 검증 실행 |
| ☐ | 검증 결과 표시 | 우측 패널 | 준비/데이터필요/문장분리필요 표시 |
| ☐ | 문제 생성 실행 | 우측 패널 | `원큐 문제 생성` 버튼 클릭 |
| ☐ | 진행 로그 확인 | 우측 패널 | 실시간 생성 로그 표시 |
| ☐ | 미리보기 결과 | 다이얼로그 | 생성된 문제 미리보기 |
| ☐ | 선택적 저장 | 다이얼로그 | 체크박스로 저장할 문제 선택 |

### 1.5 문제 관리 (문제관리 탭)
| 상태 | 테스트 항목 | 경로 | 설명 |
|:----:|------------|------|------|
| ☐ | 탭 전환 | 중앙 패널 상단 | `문제관리` 탭 클릭 |
| ☐ | 교재 선택 | 좌측 트리 | 교재 체크박스 선택 (멀티) |
| ☐ | 필터 조건 선택 | 우측 패널 | 데이터유형/문제유형/상태 필터 |
| ☐ | 필터 결과 표시 | 중앙 패널 | 필터된 지문/데이터/문제 목록 |
| ☐ | 지문 체크박스 선택 | 중앙 패널 | 삭제할 지문 선택 |
| ☐ | 전체 삭제 | 중앙 패널 | `전체삭제` 버튼 → 지문의 모든 데이터/문제 삭제 |
| ☐ | 유형별 삭제 | 중앙 패널 | `{유형}만 삭제` 버튼 → 특정 유형만 삭제 |
| ☐ | 삭제 후 새로고침 | 중앙 패널 | 삭제 후 목록 자동 갱신 |

## ⚙️ 2. 설정 탭

### 2.1 프롬프트 관리
| 상태 | 테스트 항목 | 경로 | 설명 |
|:----:|------------|------|------|
| ☐ | 프롬프트 목록 표시 | 우측 패널 | 등록된 프롬프트 목록 |
| ☐ | 프롬프트 추가 | 우측 패널 `추가` 버튼 | 새 프롬프트 폼 표시 |
| ☐ | 프롬프트 정보 입력 | 중앙 패널 | 이름, 대상, 프롬프트 내용, 출력 스키마 |
| ☐ | 변수 추출 | 중앙 패널 | `[[변수명]]` 형식 자동 인식 |
| ☐ | 프롬프트 테스트 | 중앙 패널 | 지문 선택 후 테스트 실행 |
| ☐ | 프롬프트 저장 | 중앙 패널 `저장` 버튼 | DB에 저장 |
| ☐ | 프롬프트 상태 변경 | 중앙 패널 | draft → testing → confirmed |
| ☐ | 프롬프트 삭제 | 우측 패널 | 삭제 버튼 |

### 2.2 데이터 유형 관리
| 상태 | 테스트 항목 | 경로 | 설명 |
|:----:|------------|------|------|
| ☐ | 데이터 유형 목록 표시 | 우측 패널 | 등록된 데이터 유형 목록 |
| ☐ | 데이터 유형 추가 | 우측 패널 `추가` 버튼 | 새 데이터 유형 폼 표시 |
| ☐ | 기본 정보 입력 | 중앙 패널 | 유형명, 대상 선택 |
| ☐ | 프롬프트 연결 | 중앙 패널 | 확정된 프롬프트 선택 (라이브러리 방식) |
| ☐ | 의존성 설정 | 중앙 패널 | 다른 데이터 유형에 의존 설정 |
| ☐ | 데이터 유형 저장 | 중앙 패널 `저장` 버튼 | DB에 저장 |
| ☐ | 데이터 유형 삭제 | 우측 패널 | 삭제 버튼 |

### 2.3 문제 유형 관리
| 상태 | 테스트 항목 | 경로 | 설명 |
|:----:|------------|------|------|
| ☐ | 문제 유형 목록 표시 | 우측 패널 | 등록된 문제 유형 목록 |
| ☐ | 문제 유형 추가 | 우측 패널 `추가` 버튼 | 새 문제 유형 폼 표시 |
| ☐ | 기본 정보 입력 | 중앙 패널 | 유형명, 지시문 |
| ☐ | 데이터 유형 조합 | 중앙 패널 | 필요한 데이터 유형 추가/순서 변경 |
| ☐ | 문제 유형 저장 | 중앙 패널 `저장` 버튼 | DB에 저장 |
| ☐ | 문제 유형 삭제 | 우측 패널 | 삭제 버튼 |

## 🌐 3. API 직접 테스트 (선택사항)

브라우저 콘솔 또는 Postman으로 테스트:

| 상태 | API | 메서드 | 설명 |
|:----:|-----|--------|------|
| ☐ | `/api/groups` | GET/POST | 그룹 조회/생성 |
| ☐ | `/api/textbooks` | GET/POST | 교재 조회/생성 |
| ☐ | `/api/prompts` | GET/POST | 프롬프트 조회/생성 |
| ☐ | `/api/data-types` | GET/POST | 데이터 유형 조회/생성 |
| ☐ | `/api/question-types` | GET/POST | 문제 유형 조회/생성 |
| ☐ | `/api/test-prompt` | POST | 프롬프트 테스트 (AI 호출) |
| ☐ | `/api/generate-data` | POST | 데이터 생성 (AI 호출) |
| ☐ | `/api/generate-questions` | POST | 문제 생성 (AI 호출) |
| ☐ | `/api/generate-questions/validate` | POST | 사전 검증 |
| ☐ | `/api/passages/batch-delete-generated` | POST | 일괄 삭제 |
| ☐ | `/api/status` | GET | 현황 조회 |

## ✅ 체크리스트 사용법

1. 각 항목을 테스트 후 `☐`를 `☑`로 변경
2. 이슈 발견 시 "발견된 이슈" 섹션에 기록
3. 모든 테스트 완료 후 하단에 일시 기록
